# sql-retail-sales-analysis
SQL project analyzing retail sales data to uncover insights on sales performance, customer profitability, product trends, and regional revenue using MySQL.
Retail Sales Data Analysis using SQL

The dataset consists of two main tables:

Orders Table – Contains customer, location, and order date information

Details Table – Contains product category, quantity, sales amount, and profit

Using SQL, the data is analyzed to understand sales performance, customer profitability, product trends, and regional performance.

Project Objectives

Clean and prepare raw retail sales data

Perform sales and profit analysis

Identify top-performing cities, customers, and product categories

Analyze order trends based on time (weekly, quarterly, yearly)

Practice SQL analytical techniques used by data analysts

SQL Concepts Used

This project demonstrates the use of the following SQL concepts:

Data Cleaning

Joins (INNER JOIN)

Aggregations (SUM, COUNT, AVG)

GROUP BY and HAVING

Subqueries

Window Functions (RANK)

Date Functions (YEAR, WEEK, QUARTER, DAYOFWEEK)

Data Transformation

Key Analysis Performed

Some of the important business questions answered in this project include:

Total sales generated per city

Total profit per customer

Revenue generated by each state

Category-wise revenue across payment modes

Customers with above-average profit

Top revenue generating cities within each state

Most common sub-category within each category

Top 3 revenue generating categories

Weekly order trends

Weekend vs weekday order behavior

Yearly sales performance across cities
